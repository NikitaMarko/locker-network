# Locker Network — Lambda Functions

Serverless functions for the locker rental booking system. Invoked from the backend (API) via API Gateway to handle booking business logic, notifications, and audit logging.

## Architecture

```
Backend (API)
    │
    ├── HTTP (API Gateway) ──► Booking Lambdas ──► DB (RDS)
    │                           ├── createBooking
    │                           ├── cancelBooking
    │                           ├── extendBooking
    │                           └── updateBookingStatus
    │
    ├── SQS ──► sendNotification ──► Email / SMS
    │
    └── SQS ──► writeAuditLog ──► DB (AuditLog)

CloudWatch cron (5 min) ──► expireBookings ──► DB
```

## Tech Stack

- **Runtime:** Node.js 20.x
- **Language:** TypeScript
- **ORM:** Prisma 5.22.0 (shared schema with backend)
- **IaC:** AWS SAM (`template.yaml`)
- **Infrastructure:** AWS Lambda, API Gateway, SQS, CloudWatch Events
- **Database:** PostgreSQL (Amazon RDS)

## Local Development

### 1. Start the local API Gateway

```bash
npx tsc                        # compile TypeScript → dist/
sam build                      # build SAM package
sam local start-api            # starts API Gateway on port 3000
```

SAM uses Docker to emulate the Lambda runtime — the first run pulls the image and may take a few minutes.

### 2. Test the health check endpoint

In a separate terminal:

```bash
curl.exe http://localhost:3000/health
```

Expected response:

```json
{
  "status": "ok",
  "uptime": 0,
  "services": {
    "lambda": {
      "status": "ok"
    }
  }
}
```

### 3. Running alongside Express backend

If the backend (Express) also runs on port 3000, start SAM on a different port:

```bash
sam local start-api --port 3001
```

Then set in the backend `.env`:

```
USE_LAMBDA_HEALTH=true
LAMBDA_HEALTH_URL=http://localhost:3001/health
```

### Notes

- SAM creates Docker containers on-demand per request (not persistent)
- After editing `.ts` files, re-run `npx tsc` before testing (SAM reads from `dist/`)
- On Windows PowerShell, use `curl.exe` (not `curl`, which is an alias for `Invoke-WebRequest`)

## Lambda Functions

| Function | Trigger | Uses DB | Status | Description |
|----------|---------|:---:|:---:|-------------|
| `healthCheck` | API Gateway | — | ✅ | Returns status and uptime |
| `createBooking` | API Gateway | ✅ | 🔜 | Create booking, reserve locker, audit log |
| `cancelBooking` | API Gateway | ✅ | 🔜 | Cancel booking, release locker |
| `extendBooking` | API Gateway | ✅ | 🔜 | Extend booking end time |
| `updateBookingStatus` | API Gateway | ✅ | 🔜 | Change status (admin/operator) |
| `expireBookings` | cron (5 min) | ✅ | 🔜 | Find and expire overdue bookings |
| `sendNotification` | SQS | — | 🔜 | Send email/SMS |
| `writeAuditLog` | SQS | ✅ | 🔜 | Write actions to AuditLog |

## DB Models (used by Lambdas)

- **Booking** — reservations (PENDING → ACTIVE → ENDED / CANCELLED / EXPIRED)
- **LockerBox** — locker boxes (AVAILABLE → RESERVED → OCCUPIED)
- **AuditLog** — action log for all operations
- **LockerDevice** — device status (heartbeat)
- **User** — user data for notifications (read-only)
